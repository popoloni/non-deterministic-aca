# PROMPT.md with Verification Loop

## Phase 1: Implementation
Implement the current task according to the specification.
Output your implementation between <implementation> tags.

## Phase 2: Self-Review
Review your implementation for:
- Edge cases not handled
- Security vulnerabilities
- Performance concerns
- Deviation from specification
Output your findings between <critique> tags.

## Phase 3: Final Version
If your critique identified issues, provide a corrected
implementation. If no issues were found, confirm the
implementation is final.
Output your final code between <final> tags.

Only proceed to commit after completing all three phases.
